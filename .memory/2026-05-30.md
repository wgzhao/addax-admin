2026-05-30 — 调整 table 列表页按钮布局，迁移更新操作至顶栏

概述
- 本次变更将列表页的部分操作布局进行了精简，同时把两个全局的“更新表信息”操作从页面（或分散入口）迁移到顶栏导航的“采集管理”菜单中，作为统一的全局入口。

修改文件
- `frontend/src/views/table.vue`
  - 精简并调整列表页的按钮/工具栏布局（包括批量操作显示、操作列样式等）。保留行级操作菜单与“更新表结构”项。
  - 优化了错误弹层、长文本换行与按钮视觉（若干样式调整）。
- `frontend/src/layouts/default/Topbar.vue`
  - 在顶栏“采集管理”菜单中新增两项：
    - 更新表信息 (按需) — 调用 `tableService.updateSchema({ mode: 'need' })`
    - 强制更新全部表信息 — 调用 `tableService.updateSchema({ mode: 'all' })`
  - 两个操作会使用 `notify` 显示任务启动/失败信息，便于在任意页面触发全局表信息更新。

为什么（Why）
- 减少采集表列表页的操作拥挤，将非上下文相关的全局运维操作上移到顶栏，提高可发现性与可复用性。

影响（Impact）
- 仅前端 UI/交互调整，无后端接口变更。行级的“更新表结构”仍然保留在表内操作菜单。
- 用户现在可以从任意页面通过顶栏触发表信息更新，使用方式更统一。

验证建议（Test）
1. 点击顶栏 -> 采集管理 -> “更新表信息 (按需)” / “强制更新全部表信息”，确认有通知提示任务已启动或失败信息。
2. 在表列表页检查批量操作（采集/修改/删除）与行级菜单功能是否正常。

备注
- 如需回退为页面内按钮，可在 `Topbar.vue` 中移除对应菜单项并恢复原有页面入口。
