## 前端主题与字体风格调整

- 架构决策：
  - 统一以 `frontend/src/plugins/custom-theme.ts` 作为 Vuetify 生效主题源，`theme.ts` 不再作为本轮实际生效入口。
  - 组件内尽量移除硬编码颜色，优先使用 `var(--v-theme-*)` 和 Vuetify 主题色，避免局部样式覆盖全局主题。
  - Vuetify 主题颜色最终调整为 RGB 字符串格式，兼容组件中大量 `rgb(var(--v-theme-*))` / `rgba(var(--v-theme-*), opacity)` 的使用方式。

- 修改文件和关键更改：
  - `frontend/src/plugins/custom-theme.ts`
    - 将 light/dark 主题改为参考 coinglass 的蓝色系。
    - 深色主题背景调整为 `10, 20, 40`，surface 调整为 `15, 27, 45`，primary 调整为 `59, 130, 246`。
    - 浅色主题背景调整为 `237, 245, 253`，primary 调整为 `29, 78, 216`。
  - `frontend/src/plugins/theme.ts`
    - 同步过一版主题颜色，但后续确认实际运行入口为 `custom-theme.ts`。
  - `frontend/index.html`
    - 引入 `Noto Sans SC`，改善中文小字号清晰度。
  - `frontend/src/styles/settings.scss`
    - 字体栈切换为 `Noto Sans SC` + 系统字体回退。
  - `frontend/src/styles/overrides.scss`
    - 增加字体渲染优化。
    - 将全局 hover 阴影颜色改为基于主题变量。
  - `frontend/src/views/table.vue`
    - 报错提示框改为使用 warning 主题色，不再写死黄底黄边。
  - `frontend/src/views/param.vue`
    - 删除本地表格斑马纹硬编码背景，回归全局主题样式。
  - `frontend/src/views/index.vue`
    - 标题文字颜色改为读取主题变量。
  - `frontend/src/components/task/TaskList.vue`
    - 进度文字颜色改为主题变量。
  - `frontend/src/components/dashboard/LineChart.vue`
    - 图表主线、点位、文字颜色切到主题色，并修复因主题 RGB 化导致的类型/字符串拼接问题。
  - `frontend/src/components/dashboard/L5TEtlTimeBar.vue`
  - `frontend/src/components/dashboard/L5TEtlDataBar.vue`
    - 图表标题、坐标轴、图例、tooltip 文本颜色改为主题变量。
  - `frontend/src/components/source/AddSource.vue`
  - `frontend/src/components/table/BatchAdd.vue`
  - `frontend/src/components/table/AddaxJob.vue`
  - `frontend/src/components/table/AddaxResult.vue`
  - `frontend/src/components/table/LogFiles.vue`
    - 多处表单卡片 hover 阴影从硬编码深色改为主题变量。

- 当前验证状态：
  - 通过：`bun run build:frontend`
  - 通过：Vite dev server 检测到 `custom-theme.ts` 变更并触发 page reload
  - 失败/异常：
    - 初版把主题色写成 hex，页面视觉未按预期统一，后续改为 RGB 格式。
    - 中途 `LineChart.vue` 因继续使用 `.slice()` 处理颜色导致 TS 编译失败，已修复。
    - dev server 曾异常退出，后续已重新拉起。

- 未解决风险 / 待办 / 回滚备注：
  - 仍需人工在 `http://localhost:3030` 登录后逐页确认实际视觉效果，尤其是全局背景、顶部导航、表格选中态、图表 tooltip 与登录页。
  - `theme.ts` 存在与 `custom-theme.ts` 的重复主题定义，后续应考虑删除或明确其非运行时角色，避免再次误改。
  - 图表配色目前只统一了文字和主色接入，分组柱状图的多色调色板仍保留硬编码，以保证序列可区分；若后续要完全主题化，需要设计一套主题级 chart palette。
