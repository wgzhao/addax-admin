现代主题重构与 Vuetify 开发态样式修复 — 2026-06-01

概述
- 为前端新增一套独立的现代化主题 `modern-theme`，重点提升 dark 模式下的层次感、正文可读性和中文后台的字体适配。
- 对 `table` 相关详情组件进行统一的信息架构重构，强化摘要区、操作区和内容区的主次关系。
- 修复 Vuetify 在开发态点击菜单/按钮时偶发的 `virtual:plugin-vuetify:*` Sass 预转换失败问题，同时恢复完整组件样式，避免布局错乱。

关键架构决策（永不压缩）
- 不替换现有 `custom-theme.ts`，而是新增 `frontend/src/plugins/modern-theme.ts`，通过 `frontend/src/plugins/vuetify.ts` 切换引用，降低回滚成本。
- 主题方向从“纯黑终端风”调整为“石墨蓝灰后台风”：
  - dark 模式不再使用纯黑背景；
  - 增加 `surface-raised`、`surface-overlay`、`on-surface-muted` 等语义 token；
  - 中文字体栈改为 CJK 优先，减轻 dark 模式下小字号发灰发细的问题。
- Vuetify 开发态样式链由 `styles: { configFile: ... }` 调整为 `styles: 'sass'`：
  - 保留完整组件样式；
  - 不再依赖 `virtual:plugin-vuetify:*` 的 configFile 虚拟 Sass 链；
  - 避免此前为规避报错而使用 `styles: 'none'` 导致布局全乱的问题。

动机（Why）
- dark 模式下虽然理论对比度足够，但页面实际观感仍然“文字不突出、层次不清、阅读费力”；需要从 token、字体和 surface 分层上一起修。
- `table` 相关详情组件信息堆叠过重，缺少清晰摘要层与稳定视觉节奏，需要统一重构。
- 开发态 Vuetify 的虚拟 Sass 模块偶发失效会直接阻断交互，必须从根因上修复，而不是简单关闭样式链。

本轮形成的页面规范
- 页面骨架：header card / toolbar card / table card。
- 主操作：统一放在页头右侧，使用 primary flat button。
- 表格单元格：主信息使用 `ds-cell-primary`，辅助信息使用 `ds-cell-meta` / `ds-cell-support`。
- 操作列：统一采用文字按钮，顺序优先固定为“编辑 → 上下文动作 → 删除”。
- 状态语义：success 表示启用/成功，error 表示停用/失败，primary 表示默认/当前。
- Dialog 底部：统一“取消（text）/ 保存或确认（flat）”。
- 空状态语气：统一“当前还没有配置任何 … 点击下方按钮 … 后，即可继续 …”。
