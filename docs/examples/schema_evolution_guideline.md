# Schema Evolution Guideline

说明如何让 Addax Admin 在安全模式下应用表结构演化，以及管理员在遇到复杂变更时的操作建议。

推荐流程：
1. 开启“审阅模式”：系统仅探测并记录变更，生成变更建议但不自动修改目标表。
2. 在 UI 中审阅变更建议，系统会高亮新增字段、删除字段与潜在不兼容的类型变更。
3. 对于简单变更（新增字段、兼容类型拓展），可选择“自动应用”。
4. 对于需回填或危险变更（类型不兼容、字段删除），建议创建离线迁移任务并在维护窗口内执行。

回滚策略：
- 在执行变更前，导出当前表 DDL 与 SchemaChangeLog，并在必要时用导出的 DDL 回退。
- 对涉及数据类型转换的变更，优先在影子表或新分区中验证转换结果，再整体替换。
